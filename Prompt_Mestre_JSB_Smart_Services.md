# PROMPT MESTRE — SITE JSB SMART SERVICES

Crie um site institucional e portfólio profissional **One Page**, completo, funcional e pronto para publicação no **GitHub Pages**, para a marca **JSB Smart Services**.

O site deve comunicar a união entre **Treinamento e Desenvolvimento (T&D), análise de processos, automação comercial e desenvolvimento de soluções digitais sob medida**.

Todo o conteúdo deve estar em **português do Brasil**, com linguagem corporativa, objetiva, humana e orientada a resultados.

## 1. Entrega técnica

Desenvolva o projeto utilizando:

- React com Vite;
- TypeScript;
- Tailwind CSS;
- componentes reutilizáveis;
- ícones leves, preferencialmente Lucide Icons;
- animações discretas em CSS ou Framer Motion, sem prejudicar o desempenho;
- configuração pronta para deploy no GitHub Pages;
- arquivo README com instruções de instalação, edição e publicação;
- estrutura organizada de pastas;
- nenhuma dependência paga;
- nenhum serviço externo obrigatório para o site funcionar.

O projeto deve executar com:

```bash
npm install
npm run dev
npm run build
```

Inclua configuração de `base` no Vite para facilitar a publicação em um repositório do GitHub Pages.

## 2. Identidade visual obrigatória

Utilize rigorosamente a identidade da JSB Smart Services:

- Azul Marinho Profundo: `#002B5B`;
- Ciano Vibrante: `#00E5FF`;
- Cinza Antracite: `#1A1A1A`;
- Branco: `#FFFFFF`;
- fonte principal: Inter;
- estilo minimalista, tecnológico, corporativo, elegante e limpo;
- elementos visuais inspirados em circuitos, conexões, redes, fluxos e integração entre pessoas e tecnologia.

Utilize os arquivos de marca disponibilizados:

- `jsb_logo_main.png`: aplicar somente em áreas claras;
- `jsb_logo_dark.png`: aplicar somente em áreas escuras;
- `jsb_icon_only.png`: utilizar como favicon, ícone mobile e elemento gráfico secundário.

As imagens possuem fundo incorporado. Não aplique a logo clara sobre fundo escuro nem a logo escura sobre fundo claro. Preserve proporção, cores e legibilidade. Não distorça, recolora, incline ou adicione efeitos excessivos à marca.

## 3. Direção de design

O resultado não deve parecer um template genérico.

Crie uma interface com:

- cabeçalho fixo e semitransparente;
- navegação suave entre as seções;
- Hero escuro e tecnológico;
- detalhes em ciano usados com moderação;
- fundos alternados entre branco, azul-marinho e antracite;
- linhas de circuito e pontos de conexão como elementos decorativos sutis;
- cards com bordas finas, cantos moderadamente arredondados e sombras discretas;
- títulos fortes e textos com boa hierarquia;
- bastante espaço em branco;
- animações suaves ao entrar na tela;
- botões com estados de hover, foco e clique;
- visual premium, sem excesso de neon, brilhos, partículas ou efeitos futuristas caricatos.

## 4. Cabeçalho

Criar um cabeçalho responsivo com:

- logo da JSB Smart Services;
- links: Início, Sobre, Soluções, Projetos, Método e Contato;
- botão destacado “Solicitar consultoria”;
- menu mobile em formato hambúrguer;
- indicação visual da seção ativa durante a rolagem.

## 5. Seção Hero

Criar uma seção Hero de alto impacto com fundo azul-marinho ou antracite.

### Texto principal

**Tecnologia que organiza processos. Treinamento que transforma resultados.**

### Texto de apoio

A JSB Smart Services une análise de processos, automação e Treinamento e Desenvolvimento para criar soluções digitais que simplificam rotinas, reduzem gargalos e são realmente adotadas pelas equipes.

### Botões

- “Conheça os projetos” — rolar até o portfólio;
- “Solicitar consultoria” — rolar até o contato ou abrir o WhatsApp.

### Elemento visual

Criar uma composição tecnológica abstrata baseada no símbolo da marca, circuitos, conexões e fluxos de informação. Não utilizar imagens genéricas de pessoas em escritório.

### Indicadores abaixo do Hero

Exibir quatro indicadores em destaque:

- `1.200+` horas de implantação e acompanhamento;
- `2.700+` profissionais treinados;
- `7` estados alcançados;
- `5+` anos de experiência em multipropriedade.

Tratar esses números como informações fornecidas pelo proprietário da marca. Não inventar novos indicadores.

## 6. Seção Sobre o Fundador

Título:

**Experiência de operação transformada em soluções inteligentes**

Apresentar **Jonathas da Silva Barbosa** como profissional de Marketing, Treinamento e Desenvolvimento, processos e tecnologia, com experiência prática no setor de multipropriedade.

Não destacar idade, pois essa informação torna o conteúdo rapidamente desatualizado.

### Texto sugerido

Jonathas da Silva Barbosa construiu sua trajetória atuando diretamente em operações administrativas comerciais, liderança de equipes, implantação de processos e treinamento profissional. Essa vivência permite compreender não apenas o que um sistema precisa fazer, mas como ele deve ser apresentado, implantado e incorporado à rotina das pessoas.

Ao unir T&D, análise de dados e desenvolvimento de webapps, transforma necessidades operacionais em soluções simples, funcionais e orientadas a resultados.

### Destaque de autoridade

“Participou da implantação e sustentação de iniciativas de automação comercial que somam mais de 1.200 horas de atuação e alcançaram mais de 2.700 profissionais em sete estados.”

Evite afirmar que Jonathas foi o único responsável por resultados coletivos. Utilize verbos como “participou”, “conduziu”, “apoiou” ou “liderou frentes”, conforme apropriado.

### Formação e competências

Exibir chips ou pequenos cards:

- Marketing;
- Treinamento e Desenvolvimento;
- Ciência de Dados e Inteligência Artificial;
- Automação de Processos;
- Desenvolvimento de Webapps;
- Gestão Administrativa Comercial;
- Experiência do Usuário e Adoção de Sistemas.

## 7. Seção Soluções

Título:

**Soluções que conectam pessoas, processos e tecnologia**

Criar quatro cards:

### 1. Diagnóstico de Processos

Mapeamento de fluxos, identificação de retrabalho, riscos, falhas de comunicação e gargalos operacionais.

### 2. Automação e Webapps

Desenvolvimento de MVPs, validadores, painéis, formulários inteligentes, sistemas de autorização e ferramentas para rotinas administrativas.

### 3. Implantação e T&D

Planejamento de implantação, criação de materiais, treinamento das equipes, acompanhamento da adoção e melhoria contínua.

### 4. Consultoria Sob Medida

Soluções adaptadas à realidade, ao orçamento e ao nível de maturidade tecnológica de cada empresa.

Cada card deve ter um ícone coerente, descrição curta e microinteração discreta.

## 8. Seção Portfólio de MVPs

Título:

**Projetos que transformam necessidades reais em ferramentas funcionais**

Criar uma grade responsiva de cards. Os projetos devem ser carregados a partir de um arquivo de dados separado, como `src/data/projects.ts`, facilitando futuras alterações.

Cada projeto deve conter:

- imagem ou mockup;
- categoria;
- nome;
- problema identificado;
- solução desenvolvida;
- principais recursos;
- tecnologias utilizadas;
- status: Conceito, MVP, Em validação ou Implantado;
- botão “Ver demonstração”;
- botão “Ver detalhes” ou “Repositório”.

Quando um link não tiver sido informado, deixar o botão desativado com a indicação “Disponível em breve”. Não inventar URLs.

### Projetos iniciais sugeridos

#### Automação Comercial e Validação de Processos

Sistema criado para reduzir erros de lançamento, validar regras comerciais e orientar o usuário antes da conclusão de uma operação.

#### Gestão de Autorizações

Webapp para registrar solicitações fora do padrão, encaminhar aprovações e manter histórico de decisões.

#### Gestão e Acompanhamento de Processos

Painel para organizar etapas, responsáveis, prazos, documentos e evolução de projetos operacionais.

#### Soluções Digitais Personalizadas

Conjunto de MVPs criados para resolver necessidades específicas de equipes administrativas, comerciais e de treinamento.

Não divulgar nomes de clientes, sistemas internos, dados sigilosos ou informações proprietárias sem autorização. Quando necessário, apresentar os cases de forma anonimizada.

## 9. Seção Método de Trabalho

Título:

**Da identificação do problema à adoção da solução**

Criar uma linha do tempo ou fluxo visual com cinco etapas:

1. **Escuta e diagnóstico** — entendimento do cenário, das pessoas e dos objetivos;
2. **Mapeamento de processos** — visualização do fluxo atual e identificação de gargalos;
3. **Desenho da solução** — priorização de melhorias e definição do MVP;
4. **Desenvolvimento e validação** — construção, testes e ajustes com usuários reais;
5. **Implantação e acompanhamento** — treinamento, adoção, medição e evolução contínua.

No desktop, apresentar horizontalmente. No mobile, apresentar verticalmente.

## 10. Seção Diferenciais

Criar uma seção escura com três diferenciais:

- **Visão de quem conhece a operação:** soluções baseadas em problemas reais, não apenas em tecnologia;
- **Tecnologia com adoção:** desenvolvimento acompanhado de comunicação, treinamento e suporte;
- **Construção sob medida:** ferramentas ajustadas à necessidade e ao estágio de cada negócio.

Adicionar a frase:

**Uma solução só gera resultado quando simplifica o processo e faz sentido para quem vai utilizá-la.**

## 11. Seção Contato

Título:

**Vamos transformar um gargalo em uma solução inteligente?**

Criar formulário com:

- Nome;
- E-mail;
- Telefone ou WhatsApp;
- Empresa;
- Assunto;
- Mensagem;
- aceite da política de privacidade.

Como não há backend obrigatório, implementar duas alternativas:

1. botão para gerar e abrir uma mensagem formatada no WhatsApp;
2. botão para abrir o cliente de e-mail com assunto e mensagem preenchidos.

Manter os dados de contato em um único arquivo de configuração, por exemplo `src/config/contact.ts`, com placeholders claros:

- `WHATSAPP_NUMBER`;
- `PROFESSIONAL_EMAIL`;
- `LINKEDIN_URL`;
- `GITHUB_URL`.

Não inventar telefone, e-mail ou perfis sociais.

Validar os campos no navegador, mostrar mensagens de erro acessíveis e apresentar confirmação antes de abrir o canal escolhido.

## 12. Rodapé

Incluir:

- logo para fundo escuro;
- frase curta sobre a marca;
- menu resumido;
- links sociais;
- copyright com ano automático;
- texto “JSB Smart Services — Pessoas, processos e tecnologia conectados para gerar resultados.”

## 13. Responsividade

O site deve funcionar perfeitamente em:

- smartphones a partir de 320 px;
- tablets;
- notebooks;
- monitores grandes.

Evitar rolagem horizontal. Garantir que títulos, logos, cards, botões e formulários se adaptem corretamente.

## 14. Acessibilidade

Aplicar:

- HTML semântico;
- contraste compatível com WCAG AA;
- textos alternativos nas imagens;
- navegação completa por teclado;
- estados de foco visíveis;
- `aria-label` quando necessário;
- suporte a `prefers-reduced-motion`;
- tamanho mínimo confortável para botões e links;
- formulário com labels reais, não apenas placeholders.

## 15. Performance

Aplicar:

- carregamento preguiçoso de imagens fora da primeira dobra;
- imagens otimizadas;
- divisão adequada de componentes;
- fontes carregadas de maneira eficiente;
- animações leves;
- prevenção de layout shift;
- meta de desempenho elevada no Lighthouse.

Não utilizar vídeos pesados no fundo do Hero.

## 16. SEO e compartilhamento

Configurar:

- título: `JSB Smart Services | Automação, Processos, T&D e Webapps`;
- meta description objetiva;
- palavras-chave relacionadas a consultoria em automação, análise de processos, desenvolvimento de webapps, T&D e tecnologia;
- Open Graph;
- Twitter Cards;
- favicon com o ícone da marca;
- canonical configurável;
- `robots.txt`;
- `sitemap.xml`;
- dados estruturados Schema.org dos tipos `ProfessionalService` e `Person`, sem inventar endereço ou dados de contato.

## 17. Experiência e interações

Implementar:

- rolagem suave;
- botão flutuante de WhatsApp somente após o usuário avançar na página;
- botão de voltar ao topo;
- animação sutil dos indicadores numéricos;
- filtros simples no portfólio por categoria ou status;
- modal ou página interna leve para detalhes dos projetos;
- feedback visual de carregamento, sucesso e erro;
- respeito à preferência de redução de movimento.

## 18. Conteúdo editável

Centralizar conteúdos em arquivos de dados para que o proprietário consiga alterar sem editar toda a interface:

- projetos;
- indicadores;
- serviços;
- formação e competências;
- links sociais;
- informações de contato.

Adicionar comentários no código indicando onde substituir:

- foto profissional;
- links dos projetos;
- WhatsApp;
- e-mail;
- LinkedIn;
- GitHub;
- domínio final.

## 19. Critérios de aceitação

A entrega só será considerada completa quando:

- o projeto abrir sem erros;
- todas as seções estiverem implementadas;
- menu e botões funcionarem;
- o formulário validar os campos;
- o site estiver responsivo;
- não houver links falsos;
- as logos estiverem aplicadas corretamente;
- o conteúdo estiver em português do Brasil;
- o build de produção for concluído sem erros;
- houver README de publicação no GitHub Pages;
- nenhum dado confidencial ou nome de cliente for inventado;
- não existirem textos genéricos em latim ou placeholders visíveis, exceto campos de configuração claramente identificados.

## 20. Resultado esperado

Entregue todos os arquivos do projeto, prontos para copiar para um repositório GitHub, com uma interface visualmente forte, profissional e confiável. O site deve posicionar a JSB Smart Services como uma consultoria capaz de compreender a operação, capacitar pessoas e desenvolver soluções digitais que geram resultados concretos.
